## Update Process

### 1. Identify Change Type

Map your changes to categories:
- **Added**: New features, new files, new endpoints
- **Changed**: Modifications to existing behavior
- **Deprecated**: Features marked for future removal
- **Removed**: Deleted features or code
- **Fixed**: Bug fixes
- **Security**: Security patches or vulnerability fixes

### 2. Write Clear Entries

Good entries:
- Start with a verb (Add, Fix, Update, Remove)
- Are specific and actionable
- Reference issue/PR numbers if applicable
- Explain impact to users

Examples:
- `Add user authentication with JWT tokens (#123)`
- `Fix memory leak in file upload handler (#456)`
- `Update API rate limit from 100 to 1000 requests/hour`

Bad entries:
- `Fixed stuff`
- `Updates`
- `Various improvements`
