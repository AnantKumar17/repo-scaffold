## Migration Safety Checklist

Before applying migrations, verify:

- [ ] Migration is backward compatible (can rollback)
- [ ] No data loss (dropping columns/tables with data)
- [ ] Indexes added for new foreign keys
- [ ] Default values provided for new NOT NULL columns
- [ ] Migration tested on a copy of production data
- [ ] Rollback migration tested and works
- [ ] Large table changes use batching/background jobs
- [ ] Downtime requirements documented (if any)
